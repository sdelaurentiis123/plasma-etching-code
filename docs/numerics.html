The Numerics: Representing and Moving the Surface

Level-set methods, signed distance, high-order Hamilton–Jacobi solvers, reinitialization, and sparse data structures.

1. Why a level set

The surface is stored implicitly as the zero contour of a scalar field \(\varphi(\mathbf{x},t)\):

$$\varphi>0\ \text{(solid)},\qquad \varphi<0\ \text{(gas)},\qquad \varphi=0\ \text{(surface)}.$$

Two properties make this the standard choice for etch/deposition:

  • Topology is automatic. Features merging, mask undercut pinching off, voids forming — all handled with no special-case code, because connectivity is never tracked. You evolve a field, not a curve.
  • Geometry comes from derivatives. The outward normal and curvature are $$\mathbf{n}=-\frac{\nabla\varphi}{|\nabla\varphi|},\qquad \kappa=\nabla\!\cdot\!\frac{\nabla\varphi}{|\nabla\varphi|}.$$ The minus sign puts \(\mathbf{n}\) into the gas (decreasing \(\varphi\)), where the flux comes from.

2. Signed distance and the eikonal property

Among all fields with the same zero set, we keep the signed-distance function, characterized by

$$|\nabla\varphi|=1\quad\text{(eikonal equation).}$$

This is not required by the math, but it keeps \(\nabla\varphi\) well-conditioned so normals, curvature and the upwind scheme stay accurate. A field that drifts from \(|\nabla\varphi|=1\) produces noisy normals and a smeared front.

3. The level-set evolution equation

If the surface moves with normal speed \(F\) (the etch rate from chemistry), the field obeys a Hamilton–Jacobi PDE:

$$\frac{\partial\varphi}{\partial t} + F\,|\nabla\varphi| = 0.$$
Sign check

\(\nabla\varphi\) points into the solid (\(\varphi>0\) side). With \(F>0\) the equation moves the zero-contour along \(+\nabla\varphi\), i.e. into the solid — material is removed. Since the chemistry gives \(V_n\ge0\), etching corresponds to \(F\ge0\). Consistent.

4. Spatial discretization: upwinding the Hamiltonian

The term \(|\nabla\varphi|\) cannot be central-differenced — information must propagate from the correct upwind direction or the scheme is unstable. The Godunov / Rouy–Tourin Hamiltonian for an eroding front (\(F>0\)) is

$$|\nabla\varphi|^2 \approx \max(D^-_x\varphi,0)^2+\min(D^+_x\varphi,0)^2+\;(\text{same for }y,z),$$

where \(D^\pm\) are forward/backward differences. This is first order — simple but numerically diffusive: it smears sharp corners and loses a little volume each step.

The upgrade: WENO

WENO (Weighted Essentially Non-Oscillatory) reconstructs \(\nabla\varphi\) from a high-order stencil, adaptively down-weighting sub-stencils that straddle a kink so it stays sharp without spurious oscillations. Fifth-order WENO is the standard for level-set topography (Manstetten, ViennaLS). The payoff is indirect but real: a less diffusive scheme hits target fidelity on a coarser grid, which means fewer cells, fewer triangles, and fewer rays downstream.

5. Temporal discretization and CFL

Time is advanced with TVD Runge–Kutta (Total-Variation-Diminishing, 2nd–3rd order) — convex combinations of forward-Euler stages that do not increase total variation, so the front stays monotone. The explicit step is limited by the CFL condition:

$$\Delta t \;<\; \alpha_{\rm CFL}\,\frac{\Delta x}{|F|_{\max}},\qquad \alpha_{\rm CFL}\in(0,1].$$

Physically: the front may not cross more than a fraction of a cell per step. Where \(F\) is large the timestep is sub-divided (the reference code computes nsub from exactly this ratio).

6. Reinitialization

After advection \(\varphi\) is no longer a signed distance (\(|\nabla\varphi|\neq1\)). Reinitialization restores it by re-solving the eikonal equation \(|\nabla\varphi|=1\) with the zero set fixed — either by a PDE relaxation

$$\partial_\tau\varphi+\operatorname{sgn}(\varphi_0)\big(|\nabla\varphi|-1\big)=0$$

to steady state, or directly by the fast marching method (a Dijkstra-like \(O(N\log N)\) sweep). The reference code reinitializes every step with fast marching.

Bias contributor #5 — numerical diffusion

First-order upwind plus reinitializing every step is doubly diffusive, and 32 cells across an 8 µm trench (\(\Delta x=0.25\)) is coarse. Fixes: WENO + TVD-RK; reinitialize lazily (every \(k\) steps) and locally (only near the moving front); refine \(\Delta x\). Each reduces diffusion and the depth/shape bias.

7. Velocity extension

The etch rate \(V_n\) is defined only on the surface, but the PDE runs on the whole grid. \(F\) must be extended off the surface into a band. The principled extension makes \(F\) constant along normals, which preserves the signed-distance property under advection:

$$\nabla F\cdot\nabla\varphi = 0,$$

solved by fast marching outward from the front, or by a closest-point lookup \(F(\mathbf{x})=V_n(\mathbf{x}_{\rm cp})\). The reference code uses a cheap nearest-segment KD-tree query — an approximation worth upgrading.

8. Sparse data structures (the 3D enabler)

A dense 3D grid costs \(O(n^3)\) memory for a surface that is intrinsically 2D — nearly all of it wasted. Two SOTA structures fix this:

This is the difference between feasible and impossible for a 1000-layer channel hole. It also pairs directly with DDA grid traversal for ray tracing — see Flux & Transport.

9. Surface extraction and accumulation

For chemistry and (optionally) ray tracing you need an explicit surface. Marching cubes (2D: marching squares) extracts the \(\varphi=0\) isosurface as a triangle mesh; each element carries a normal, area and centroid. A lighter alternative is a disk mesh — surface sample points with normals and areas but no connectivity — which avoids rebuilding mesh topology every step as the surface moves. ViennaPS uses disk meshes for flux accumulation precisely for this reason.

The numerics, in one line

Implicit field for free topology and geometry; high-order non-diffusive Hamilton–Jacobi solver (WENO + TVD-RK) for the motion; reinitialization to keep distances clean; sparse storage to make 3D tractable; explicit (disk/triangle) mesh only where you need to touch the surface.
