The Cross-Scale Fast Model: From Atoms to Reactor

How to extend Graves' atomistic simulations into a learned surface operator, and chain differentiable surrogates across every scale into one fast, invertible model.

The end state: a single differentiable pipeline where you turn a reactor knob and watch a 3D feature profile change — and, run backward, you name a target profile and it tells you the recipe. The atomistic physics (Graves) grounds the surface response; the plasma physics (Kushner) supplies the inputs; the feature solver sits in the middle. The trick to making it fast is to replace each expensive scale with a learned operator while keeping the geometry physical.

The wall

These scales differ by ~15 orders of magnitude in time. Molecular dynamics resolves femtoseconds; a feature etches over seconds. You can never run them on one clock. Classically the fix is one-way, offline coupling: run HPEM once, hand fluxes to the feature code; fit the surface model to MD once, by hand. The opportunity is to replace each scale-bridge with a fast differentiable surrogate, so the scales can be coupled tightly and inverted end-to-end.

2. Why MD can never be inline (the amortization argument)

Back-of-envelope

A feature has \(10^4\)–\(10^6\) surface elements. Each receives \(\gtrsim10^3\) particle impacts per timestep, over \(\sim10^2\) steps: \(\gtrsim10^9\) impacts per simulation. One MD impact is \(\sim\)ns of dynamics on \(\sim10^3\)–\(10^4\) atoms — seconds to minutes of compute each. Inline MD would be \(\sim10^{15}\times\) too slow. MD's role is therefore to be amortized into a surrogate: run it offline across the input space, train a model, evaluate the model inline.

4. The surface-response operator \(\mathcal{S}\) — the linchpin

Define one operator that the whole field can reuse:

$$\mathcal{S}:\;\big(\underbrace{\text{species},E,\theta}_{\text{incident particle}},\;\underbrace{\mathbf{s}}_{\text{local surface state}}\big)\;\longmapsto\;\big(\underbrace{Y}_{\text{removal}},\;\underbrace{\boldsymbol\Phi_{\rm out}}_{\text{emitted/product fluxes}},\;\underbrace{\sigma}_{\text{sticking}},\;\underbrace{\Delta\mathbf{s}}_{\text{state update}},\;\underbrace{\text{roughness moments}}_{\text{stochastic}}\big).$$
Why this one object is the whole game
  • It replaces inline MD — fast enough to call per surface element per step.
  • It replaces the phenomenological chemistry (Langmuir + Steinbrüchel) — so it directly retires bias contributor #1, the dominant one. Higher fidelity and faster, at once.
  • It is reusable across feature codes — MCFPM, ViennaPS, ours all need exactly this. The operator is the defensible, portable artifact: the core of the "plasma-chemistry foundation model."

How to train it without drowning in MD

  • Residual learning (control variate for ML). Don't learn \(Y\) from scratch — learn the correction to the known phenomenological model: \(Y=Y_{\rm Steinbr\ddot{u}chel}+\mathcal{S}_\Delta(\cdot)\). The network only carries what physics already doesn't, so it needs far less data and can't violate known asymptotics badly.
  • Bake in physics constraints. The \(\sqrt{E}\) threshold behavior, rotational symmetry in azimuth, the zero-below-threshold floor — impose these as hard structure, not things to be learned.
  • Active learning for the input-space curse: sample MD where the operator is most uncertain (uncertainty-quantified MLIPs make this measurable), not on a blind grid.
  • Output a distribution, not a mean. MD impacts are stochastic; the same input gives a spread of outcomes. The operator predicts moments, which is also where physical roughness comes from (distinct from Monte-Carlo noise — see Flux & Transport §3).

5. The plasma surrogate \(\mathcal{P}\) — Kushner's layer, amortized

$$\mathcal{P}:\;(\text{power},\,p,\,\text{gas mix},\,\text{bias waveform},\,\text{geometry})\;\longmapsto\;(\boldsymbol\Gamma,\,\text{IED},\,\text{IAD})\ \text{at the wafer}.$$

Trained on HPEM/PCMCM runs (or, before that access, on published IED/IAD parametrizations). Neural-operator surrogates for plasma are already demonstrated for plasma-edge and MHD codes, so the architecture class is proven on plasma data. \(\mathcal{P}\) replaces inline HPEM and makes the reactor knobs differentiable inputs to the whole chain.

6. The coupled differentiable stack

The payoff: inverse design across scales

Because the chain is end-to-end differentiable, \(\partial(\text{profile})/\partial(\text{reactor knobs})\) exists. You can pose: "I want this 3D-NAND channel-hole shape — what power, pressure, gas mix, and bias waveform produce it?" and solve it by gradient descent. That is the holy grail of process design, and no shipping tool does it because no shipping tool is differentiable across scales.

7. Making the timescale coupling consistent

With the wall of §1, how do scales coexist in one run? Timescale separation (the heterogeneous multiscale method): each scale assumes the others are quasi-static within its own step. The surrogates make this clean — instead of waiting for a slow inner simulation to converge, the operator returns the converged response instantly. This is the same operator-splitting-in-time the feature solver already does when it freezes the surface during a flux calculation (see Physics §8), now applied up and down the scale ladder. Coupling can be sequential (cheap, small lag error) or concurrent self-consistent (a fixed point per macro-step) where accuracy demands.

The one-paragraph vision

Keep the geometry physical (level set, ballistic transport, radiosity — all differentiable), and make the physics inputs learned operators: \(\mathcal{P}\) for the plasma (from Kushner-class data), \(\mathcal{S}\) for the surface response (from Graves-class MD). Train each against its own ground truth, validate the coupled chain against experiment, and differentiate the whole thing so a target shape yields a recipe. That is one fast, accurate, invertible model from atoms to reactor — and the surface operator \(\mathcal{S}\) is the reusable core that the rest of the field would adopt.
