Differentiable, GPU-Accelerated 3D Feature-Scale Plasma-Etch Simulation

A technical explainer: the field, the physics, the numerics, the acceleration, and where the defensible novelty actually lives.

In a hurry? What petch does — one page: the validated claims, the gates, and the honest limits, with links to every reproducer.

We are building petch, a simulator that predicts how a single semiconductor feature — a trench, a contact hole, a 3D-NAND channel hole — changes shape as plasma etches it. The target is three-dimensional, GPU-accelerated, differentiable, and ultimately surrogate-accelerated. This site explains every layer of that stack from first principles.

The one idea to hold onto

Almost every optimization in this project is the same move wearing different clothes: do not spend compute where there is no information. A near-collimated ion beam carries almost no angular variance, so it needs almost no rays. A diffuse neutral field is deterministic at equilibrium, so it should be solved as linear algebra, not sampled as noise. Reducing variance makes the simulation faster, smoother, and differentiable — all at once.

The two problems, kept strictly separate

The single most important conceptual move is to refuse to talk about "accuracy" as one thing. There are two completely different error sources, with two different fixes:

ProblemWhat it isWhat fixes it
Statistical noise (variance)Monte-Carlo sampling roughness. Each surface element collects only tens of particle hits, so the flux estimate carries \(\sim 1/\sqrt{N}\approx 10\text{–}20\%\) noise. It averages to the right answer but looks jagged and corrupts gradients.Variance reduction: quasi-Monte-Carlo, control variates, radiosity, more rays. Exact — no fidelity cost. See Acceleration.
Model-form & numerical biasA systematic offset (e.g. \(\sim 7\%\) in depth) from approximations in the physics model and the discretization — not from sampling. It does not average away.Better models & schemes: exact coverage chemistry, integrate yield over the energy spectrum, higher-order advection, ion reflection. See Physics and Numerics.

Conflating these is what makes "accuracy" feel mushy. Noise is cured by counting; bias is cured by modeling. They are measured on different benchmark axes (rays-to-fidelity vs. Chamfer-distance).

The honest thesis

The incumbent open-source tool (ViennaPS) already has a maturing GPU/OptiX ray tracer. The industry tool (Sentaurus) already solves re-emission by radiosity, by default. So "we ray-trace faster on the same algorithm" is not a win — RT cores are RT cores. The three things nobody ships together, and that this project can:

  1. Equal-or-better fidelity at far fewer rays — QMC + control variates + the ion/neutral solver split, now including a deterministic, noise-free Sn neutral transport that reproduces ViennaPS ARDE (see ViennaPS Validation §7). An algorithmic win, not a hardware one.
  2. Differentiability — gradients of the final profile with respect to recipe knobs, enabling inverse design (target shape → process recipe).
  3. Learned operators — surrogate the two expensive pieces (stiff surface chemistry, multiple-scattering re-emission) with neural operators trained on high-ray ground truth. The same machinery as a plasma-chemistry foundation model.

Status of the reference code

The original proof-of-concept (feature_etch.py) is a 2D simulator that matches ViennaPS in shape and ARDE trend parameter-free, needing only a single global rate_scale (≈0.29 in that 2D PoC) for absolute depth — an absolute-units constant, the analog of ViennaPS's unitConversion, whose value tracks the flux-normalization convention (see Physics Grounding), not a fudge. The 3D engine (petch) now replicates ViennaPS's ballistic model directly — matching on holes, ~0.06–0.1 gentler on trenches (ViennaPS Validation) — so the six historical bias contributors are now second-order; the first-order residual is the structural ballistic-vs-wafer gap (gas conductance / charging). The plan page tracks them.
